# Output Guide

This document explains the main output files generated by pipeline_v3.

============================================================
1. Full-route mode
============================================================

Run:

    python run_pipeline.py --config configs/full_route_stage3.yaml

The preferred final outputs are:

    final_top_routes.md
    final_top_routes.csv

Typical path:

    outputs/inference/<infer_name>/routes_flow_fallback_retrieval_baseline_element_reranked/final_top_routes.md
    outputs/inference/<infer_name>/routes_flow_fallback_retrieval_baseline_element_reranked/final_top_routes.csv

These files contain the final route-level recommendations after Stage35 reranking.

============================================================
2. Best route per precursor
============================================================

Files:

    synthesis_routes_stage35_v21_best_per_precursor.md
    synthesis_routes_stage35_v21_best_per_precursor.csv

These files keep one best condition for each precursor set.

They are useful for comparing different precursor combinations, but they are not always identical to the global final route ranking.

============================================================
3. Stage35 reranked routes
============================================================

Files:

    synthesis_routes_stage35_v21_hybrid_reranked.md
    synthesis_routes_stage35_v21_hybrid_reranked.csv

These files contain the route-level ranking before the final top-route export.

They are useful for checking how Stage35 scores all candidate routes.

============================================================
4. Readable routes before Stage35
============================================================

Files:

    synthesis_routes_readable.md
    synthesis_routes_readable.csv

These files summarize the Stage3 condition predictions before final Stage35 reranking.

They are useful for debugging Stage3 outputs, but they should not be treated as the final recommendation.

============================================================
5. Display-filtered routes
============================================================

Files:

    synthesis_routes_display_filtered.md
    synthesis_routes_display_filtered.csv

These files remove routes with unreasonable temperature or time ranges before Stage35 reranking.

============================================================
6. Precursor-only mode
============================================================

Run:

    python run_pipeline.py --config configs/precursor_only.yaml

Final outputs:

    precursor_only_recommendations.md
    precursor_only_recommendations.csv

Typical path:

    outputs/inference/<infer_name>_precursor_only/precursor_only/precursor_only_recommendations.md
    outputs/inference/<infer_name>_precursor_only/precursor_only/precursor_only_recommendations.csv

Use these files when only precursor sets are needed.

============================================================
7. Important Stage2 intermediate files
============================================================

Raw GFlowNet samples:

    test_samples.csv

Composition-constrained samples:

    test_samples_composition_constrained.csv

Unique summarized Stage2 candidates:

    unique_sets_ranked.csv

Fallback-completed candidates:

    unique_sets_ranked_with_fallback.csv

Retrieval-augmented candidates:

    retrieval_npz_candidates.csv

Merged Stage2 candidates:

    unique_sets_ranked_with_fallback_retrieval_baseline.csv

Final Stage2 candidate table:

    unique_sets_ranked_with_fallback_retrieval_baseline_element_reranked.csv

This final Stage2 file is the main precursor candidate table used by downstream Stage3 and Stage35.
