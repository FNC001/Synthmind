Structure-to-Synthesis Route 批量运行说明
========================================

1. 程序整体功能
----------------
该程序用于对一批晶体结构文件 POSCAR 进行自动化合成路径推测。整体流程包括：

输入 POSCAR 结构
→ 单结构 pipeline 推理
→ Stage2 前驱体组合预测
→ Stage3 合成条件预测
→ Stage35/V43 路线重排序
→ route-level 置信度分级
→ batch-level 结果汇总
→ reliability 可靠性检查
→ 最终结构-路径-证据表输出

最终输出会回答：

- 每个结构的化学式是什么？
- 推荐的合成前驱体组合是什么？
- 该合成路径是如何获得的？
- 对应评分是多少？
- 路线置信等级如何？
- 是否需要人工复核？


2. 输入文件准备
----------------
批量输入 POSCAR 放在：

/Users/wyc/SynPred/data/infer/batch_500_poscars/

推荐每个结构一个文件夹，格式如下：

batch_500_poscars/
├── sample_000001/
│   └── POSCAR
├── sample_000002/
│   └── POSCAR
├── sample_000003/
│   └── POSCAR
...

每个 POSCAR 建议为标准 VASP5 格式：

第 6 行：元素符号
第 7 行：元素数量

例如：

Li Fe P O
2 1 1 4

程序会从 POSCAR 中自动解析：

target_formula
target_elements
target_natoms


3. 第一层：批量主推理
----------------------
进入目录：

cd /Users/wyc/SynPred/scripts/07_infer/structure_to_synthesis_route/batch_adaptive

运行完整批量推理：

python run_adaptive_batch_pipeline.py \
  --config configs/batch_500.yaml \
  --mode run_and_audit

如果只想测试前几个结构：

python run_adaptive_batch_pipeline.py \
  --config configs/batch_500.yaml \
  --mode run_and_audit \
  --limit 5

如果只想跑某一个结构：

python run_adaptive_batch_pipeline.py \
  --config configs/batch_500.yaml \
  --mode run_and_audit \
  --case_id_filter case_000001_sample_000001_POSCAR \
  --force

第一层输出目录：

/Users/wyc/SynPred/outputs/batch_adaptive/batch_500/

核心输出：

master_status.csv
master_status_standard.csv
batch_report.md

logs/

其中 master_status_standard.csv 是每个结构的 case-level 状态汇总。


4. 第二层：一键后处理与可靠性检查
----------------------------------
完成第一层后，推荐直接运行总入口：

cd /Users/wyc/SynPred/scripts/07_infer/structure_to_synthesis_route

bash run_batch_full_stack.sh /Users/wyc/SynPred batch_500 5 dry_run

参数含义：

第 1 个参数：PROJECT_ROOT
第 2 个参数：BATCH_NAME
第 3 个参数：每个结构导出的 top N 路线
第 4 个参数：reliability 模式，dry_run 或 execute

常用命令：

bash run_batch_full_stack.sh /Users/wyc/SynPred batch_500 5 dry_run

如果后续确认要执行恢复动作：

bash run_batch_full_stack.sh /Users/wyc/SynPred batch_500 5 execute

9. 当前评分含义
----------------
最终表里的评分不是实验验证结果，而是模型内部排序与一致性分数。

主要包括：

final_recommendation_score
  综合推荐分数，用于最终排序。

stage35_v43_safe_strict_score
  Stage35/V43 safe-strict 路线重排序分数，反映路线整体合理性。

real_stage3_condition_reference_support_score
  Stage3 条件参考支持分数，表示预测条件与已学习条件分布的一致性。

condition_distribution_support_score
  条件分布支持分数，作为 Stage3 条件合理性的辅助指标。

route_refined_status
  路线级别置信标签，包括：

route_high_confidence
route_medium_confidence
route_low_confidence
route_needs_manual_check

推荐优先使用：

route_high_confidence
route_medium_confidence
